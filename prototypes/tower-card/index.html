Tower card — five in-context ideas

The full board page is identical in every variant (real Hangar tokens, fictional “copperline” data mirroring the live page’s states); only the tower card between the project header and the flights list changes. Each page also shows the same card in its behind-origin warning state at the bottom. Flip variants with the floating pill nav.

  1. 01Identity “Tower” becomes a nameplate — title, “main worktree — where merged work lands”, branch — with the familiar labeled cells kept to the right.
  2. 02Row zero The tower sits on the flights table’s own column grid — env under Env, reported under Reported — while keeping its panel chrome.
  3. 03Sentence Leads with the tower’s condition as a sentence — “Up to date with origin — merged work is running at :3000.” — facts in one quiet line; the card tints amber when behind.
  4. 04Clusters Facts grouped under Checkout / Environment / Release labels with hairline dividers, so the card has internal logic instead of six flat cells.
  5. 05Env-forward The right end is a bordered “:3000 open dev server” button — one obvious click to the merged app; the pull-to-update warning lands right beside it.
