# 2.3 Adapter Command

The `flight-rules adapter` command generates agent-specific adapter files.

## Goals

- Generate adapter files for different AI coding tools
- Support Codex (AGENTS.md + repo-local skills), Cursor (AGENTS.md + commands), and Claude Code (CLAUDE.md + commands)
- Allow generating adapters after initial installation

## Constraints

- Adapters go in project root (user-owned space)
- Should detect existing adapters and prompt before overwrite
- Codex adapter requires copying repo-local skills to `.agents/skills/`
- Claude adapter requires copying commands to `.claude/commands/`

## Dependencies

- **Requires**: Existing Flight Rules installation
- **Enables**: Tool-specific integration

---

## 2.3.1. Shared AGENTS Adapter

**Goal**: Generate a shared `AGENTS.md` for Codex and Cursor

**Approach**:
- Copy adapter template to project root as `AGENTS.md`
- Template points to `.flight-rules/AGENTS.md` for guidelines
- References `docs/` for project documentation
- Points to `.flight-rules/commands/` as the source of truth
- Mentions optional Cursor slash commands and Codex skills if installed

**Acceptance Criteria**:
- `AGENTS.md` created at project root
- Contains pointers to Flight Rules structure
- Codex and Cursor can discover project conventions

**Status**: ✅ Complete

---

## 2.3.2. Codex Adapter

**Goal**: Generate repo-local Codex skills

**Approach**:
- Reuse the shared `AGENTS.md` adapter
- Create `.agents/skills/`
- Copy Flight Rules skill folders from `.flight-rules/skills/codex/`

**Acceptance Criteria**:
- `.agents/skills/` contains Flight Rules skills
- Skills route Codex to `.flight-rules/commands/` workflows
- Codex users can discover Flight Rules workflows without Claude-specific command mirroring

**Status**: ✅ Complete

---

## 2.3.3. Cursor Adapter

**Goal**: Generate native Cursor slash commands

**Approach**:
- Reuse the shared `AGENTS.md` adapter
- Create `.cursor/commands/`
- Copy command files from `.flight-rules/commands/`

**Acceptance Criteria**:
- `.cursor/commands/` contains all workflow commands
- Cursor users get slash command access without a separate root adapter file

**Status**: ✅ Complete

---

## 2.3.4. Claude Code Adapter

**Goal**: Generate `CLAUDE.md` and slash commands for Claude Code

**Approach**:
- Copy adapter template to project root as `CLAUDE.md`
- Create `.claude/commands/` directory
- Copy command files from `.flight-rules/commands/`

**Acceptance Criteria**:
- `CLAUDE.md` created at project root
- `.claude/commands/` contains all workflow commands
- Commands available as `/dev-session.start`, etc.

**Status**: ✅ Complete

---

## 2.3.5. Adapter Flags

**Goal**: Support command-line flags for non-interactive generation

**Approach**:
- `--codex` flag generates Codex adapter
- `--cursor` flag generates Cursor adapter
- `--claude` flag generates Claude Code adapter
- Both flags can be combined
- No flags triggers interactive prompt

**Acceptance Criteria**:
- Flags work in non-interactive environments
- Can generate specific adapters without prompts

**Status**: ✅ Complete
