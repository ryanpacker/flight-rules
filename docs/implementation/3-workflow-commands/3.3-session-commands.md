# 3.3 Session Commands

Commands for structured coding sessions with documentation.

## Goals

- Provide start/end ritual for focused coding work
- Capture context, goals, and learnings
- Update progress and implementation docs

## Commands

- `dev-session.start` — Begin a coding session
- `dev-session.end` — End a session and document results

## Dependencies

- **Requires**: Implementation specs for structured work
- **Enables**: Progress tracking, context preservation

---

## 3.3.1. dev-session.start Command

**Goal**: Begin a structured coding session with clear goals

**Approach**:
- Review recent progress and current state
- Identify tasks to work on
- Create session plan with specific goals
- Prepare context for focused work

**Key Features**:
- Reads progress.md and implementation specs
- Helps select tasks from existing specs
- Creates session focus document
- Sets up context for productive session

**Acceptance Criteria**:
- Session has clear, specific goals
- Tasks are identified from implementation specs
- Context is established for the work ahead

**Status**: ✅ Complete

---

## 3.3.2. dev-session.end Command

**Goal**: Document session results and update project state

**Approach**:
- Summarize work completed
- Update task statuses in implementation docs
- Add entry to progress.md
- Capture critical learnings if any
- Create session log file

**Key Features**:
- Generates session summary
- Updates Status fields in task specs
- Appends to progress.md
- Creates `docs/session_logs/YYYYMMDD_HHMM_session.md`

**Acceptance Criteria**:
- Session log captures what was done
- Implementation docs reflect new statuses
- Progress.md has new entry
- Learnings captured if significant

**Status**: ✅ Complete
