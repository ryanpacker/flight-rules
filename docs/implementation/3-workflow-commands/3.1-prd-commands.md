# 3.1 PRD Commands

Commands for creating and refining Product Requirements Documents.

## Goals

- Guide users through structured PRD creation
- Support both conversational exploration and quick one-shot generation
- Ensure PRDs have measurable goals and clear non-goals

## Commands

- `prd.create` — Create a new PRD
- `prd.clarify` — Refine specific sections of an existing PRD

## Dependencies

- **Requires**: None
- **Enables**: Implementation planning (`impl.outline`, `impl.create`)

---

## 3.1.1. prd.create Command

**Goal**: Create a comprehensive Product Requirements Document

**Approach**:
- Adopt persona of senior product manager
- Walk through sections: Overview, Goals, Non-Goals, User Stories, Constraints, Success Criteria
- Push back on vague requirements
- Demand measurable success criteria

**Key Features**:
- Conversational mode: Interview user through each section
- One-shot mode: Generate draft from description, then refine
- Checks for existing PRD and offers replace/refine options

**Acceptance Criteria**:
- Creates `docs/prd.md` with all sections
- Goals are specific and measurable
- Non-goals are explicitly defined
- User stories follow "As a [user], I want [goal] so that [benefit]" format

**Status**: ✅ Complete

---

## 3.1.2. prd.clarify Command

**Goal**: Refine specific sections of an existing PRD

**Approach**:
- Read existing PRD
- Allow user to select sections to clarify
- Deep-dive on selected sections with probing questions
- Propose updates with previews

**Key Features**:
- Section selection (goals, non-goals, user stories, etc.)
- Preserves sections not being modified
- Shows diff-style preview before applying

**Acceptance Criteria**:
- Only modifies selected sections
- Previews changes before writing
- Maintains PRD structure

**Status**: ✅ Complete
