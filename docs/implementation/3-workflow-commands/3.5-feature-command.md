# 3.5 Feature Command

Command for adding features to existing projects.

## Goals

- Support mid-project feature addition
- Update both PRD and implementation docs in sync
- Check for conflicts with existing goals and non-goals

## Commands

- `feature.add` — Add a new feature to PRD and implementation docs

## Dependencies

- **Requires**: Existing PRD (`docs/prd.md`)
- **Recommended**: Existing implementation outline
- **Enables**: Incremental project growth

---

## 3.5.1. feature.add Command

**Goal**: Incorporate new features into existing project documentation

**Approach**:
- Read existing PRD and implementation structure
- Understand the new feature through clarifying questions
- Check for conflicts with non-goals and constraints
- Determine placement (existing area vs new area)
- Propose PRD updates (goals, user stories, success criteria)
- Propose implementation updates (area, task groups, tasks)
- Apply changes with user confirmation

**Key Features**:
- Conversational mode: Guide through feature definition
- One-shot mode: `/feature.add dark mode toggle`
- Conflict detection against non-goals
- Placement recommendation with rationale

**Acceptance Criteria**:
- PRD updated with new goal and user stories
- Implementation docs updated or new area created
- No silent conflicts with non-goals
- Preview before any changes applied

**Status**: ✅ Complete
