# 3.4 Testing Commands

Commands for adding tests and assessing test coverage.

## Goals

- Guide systematic test creation
- Analyze existing test coverage
- Identify testing gaps

## Commands

- `test.add` — Add tests for specific functionality
- `test.assess-current` — Analyze existing test coverage

## Dependencies

- **Requires**: Existing code to test
- **Enables**: Confidence in code changes, regression prevention

---

## 3.4.1. test.add Command

**Goal**: Add tests for specific functionality

**Approach**:
- Understand what needs testing
- Identify appropriate test types (unit, integration, e2e)
- Generate test files following project conventions
- Ensure tests are meaningful, not just coverage padding

**Key Features**:
- Analyzes code to understand testable behavior
- Follows existing test patterns in the project
- Creates focused, maintainable tests

**Acceptance Criteria**:
- Tests cover specified functionality
- Tests follow project conventions
- Tests are meaningful and maintainable

**Status**: ✅ Complete

---

## 3.4.2. test.assess-current Command

**Goal**: Analyze existing test coverage and identify gaps

**Approach**:
- Survey existing test files
- Map tests to source code
- Identify untested or under-tested areas
- Prioritize testing recommendations

**Key Features**:
- Discovers test files and frameworks
- Analyzes coverage (if tools available)
- Suggests high-priority testing targets

**Acceptance Criteria**:
- Provides overview of current testing state
- Identifies coverage gaps
- Prioritizes recommendations

**Status**: ✅ Complete
